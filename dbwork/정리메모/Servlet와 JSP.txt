Servlet 과 JSP
: Server Side언어
: 기능은 같다

여기서 서버란 WAS(Web Application Server):
	Tomcat9설치

Servlet : 기본 구조가 자바클래스
	:기본적으로 자바 로직을 짜기 편함
	:자바스크립트나 html이 가능은 하나 사용하기 불편함

JSP(Java Server Page):기본 구조가 HTML5구조
	:디자인을 하기 위한 html이나 자바스크립트를 사용하기는 편하지만
	자바로 로직처리를 하기는 자바클래스로 하는것 보다 불편함
	(실행을 하면 일단 Servlet으로 변환 후 컴파일된 class파일을 실행해서
	브라우저에 출력하는 방식)

그러다보니 로직과 디자인을 같이 처리하는 model1 방식보다는
로직을 Servlet에서 처리하고 디자인은 jsp에서 처리하는 model2 방식을 많이 사용

스프링 프레임워크나 스프링부트는 서블릿으로 만들어진 프레임워크이다.(MVC프레임워크 라고한다)
